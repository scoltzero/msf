# 爱快 iKuai 接入 msf

把爱快主路由的 DNS 与 FakeIP 流量引导到 msf 主机。

## 示例环境

| 角色 | 地址 |
|---|---|
| iKuai 网关 | `192.168.1.1` |
| msf 主机 IPv4 | `192.168.1.2` |
| msf 主机 IPv6 | `fd00::2` |

## 第 1 步：DHCP 下发 DNS

进入 **网络设置 → 内网设置 → DHCP 服务端（地址池）**，编辑对应地址池：

- **首选 DNS**：`192.168.1.2`（msf 主机 IP）
- **备用 DNS**：留空（不要填运营商/公共 DNS，否则部分客户端会绕过 msf）

## 第 2 步：FakeIP 静态路由（核心）

进入 **网络设置 → 路由设置 → 静态路由**，新增以下路由：

| 目的地址/掩码 | 网关 | 备注 |
|---|---|---|
| `28.0.0.0/8` | `192.168.1.2` | msf FakeIP v4 |
| `f2b0::/18` | `fd00::2` | msf FakeIP v6 |

> 爱快不同版本（3.X / 4.X）的菜单位置略有差异：3.X 在「网络设置 → 静态路由」，4.X 在「网络配置 → 静态路由」。条目较多时可整理成 CSV 后用「导入」功能批量添加；导入前把模板里的占位 IP 全部替换成你的 `msf主机IPv4` / `msf主机IPv6`。

## 第 3 步：可选补充路由

如需纳管硬编码公共 DNS 的设备，可再加：
`8.8.8.8/32`、`8.8.4.4/32`、`1.1.1.1/32`、`1.0.0.1/32` → `192.168.1.2`（可选）。

## 验证

客户端执行：

```bash
nslookup google.com
```

结果应落在 `28.0.0.0/8` 范围内，说明 DNS 已经过 msf。再用 `dig AAAA google.com` 验证 IPv6 结果落在 `f2b0::/18`。
